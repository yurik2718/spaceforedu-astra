# SpaceForEdu — лендинг

Продающий сайт Space for Edu на Astro + React + Tailwind, деплой на Cloudflare Pages. Три языка: `es` (по умолчанию), `en`, `ru`.

---

# ⚠️ ВНИМАНИЕ: ЧТО СДЕЛАТЬ ПЕРЕД ЗАПУСКОМ

> **Сайт НЕ готов к показу клиентам и рекламе, пока не выполнены пункты ниже.**
> Технически он собирается и открывается — но с плейсхолдерами в контактах и юридических данных. Запускать рекламу в Meta/Google без этого нельзя (откажут на модерации) и это нарушение испанского закона LSSI-CE.

## 🔴 1. ЗАПОЛНИТЬ РЕАЛЬНЫЕ КОНТАКТЫ

**Где:** Cloudflare Pages → ваш проект → **Settings → Environment variables** → Production.
**Плюс локально:** в файле `.env` (копируется из `.env.example`).

Три переменные, **все три обязательны**:

| Переменная | Что туда | Пример |
| --- | --- | --- |
| `PUBLIC_CONTACT_WHATSAPP` | WhatsApp-номер, **только цифры**, со страновым кодом, **без `+` и пробелов** | `34663689393` |
| `PUBLIC_CONTACT_EMAIL` | Контактный e-mail на вашем домене | `hola@spaceforedu.com` |
| `PUBLIC_SITE_URL` | Абсолютный URL сайта без завершающего `/` | `https://spaceforedu.com` |

Сейчас в `.env` стоит плейсхолдер `34600000000` — **это не рабочий номер**. Все кнопки WhatsApp ведут в пустоту, пока не подставите настоящий.

После подмены значений на проде — **пересоберите деплой** в Cloudflare (он не подхватывает новые env-vars на существующий билд автоматически).

## 🔴 2. ЗАПОЛНИТЬ ЮРИДИЧЕСКИЕ ДАННЫЕ (Aviso Legal)

Для Испании обязателен раздел «Aviso Legal» (закон LSSI-CE, ст. 10): название юрлица, CIF/NIF, адрес, данные реестра. Без этого:
- Meta Ads и Google Ads **отклонят модерацию** рекламы.
- На сайт можно получить штраф (от €150 до €30 000 по LSSI-CE).

**Где заполнять:** во всех трёх файлах переводов —
`src/lib/i18n/es.json`, `src/lib/i18n/en.json`, `src/lib/i18n/ru.json`.

Найдите блок `"legal_notice"` → ключ `"provider_body"`. Сейчас там стоит заглушка:

```json
"provider_body": "Space for Edu. Datos del prestador del servicio en proceso de actualización..."
```

Замените на реальные данные, например:

```json
"provider_body": "Space for Edu SL. CIF: B12345678. Calle Ejemplo 123, 28001 Madrid. Inscrita en el Registro Mercantil de Madrid, Tomo 12345, Folio 67, Hoja M-890123."
```

Пишите на каждом из трёх языков. Проверить после изменений — открыть `/es/aviso-legal/`, `/en/aviso-legal/`, `/ru/aviso-legal/`.

## 🟡 3. ЗАМЕНИТЬ БРЕНДОВЫЕ КАРТИНКИ (опционально)

Сейчас сгенерированы временные:
- `public/favicon.svg`, `public/favicon.ico`, `public/apple-touch-icon.png`, `public/icon-192.png`, `public/icon-512.png` — иконка-ракета на фирменном градиенте.
- `public/og.jpg` — 1200×630 баннер для превью при шаринге в WhatsApp / соцсетях.

Если дизайнер сделает финальные версии — просто положите файлы с теми же именами в `public/`, остальное подхватится автоматически.

## ✅ Чеклист перед первым показом клиенту / запуском рекламы

- [ ] В Cloudflare Pages заданы все три переменные `PUBLIC_*` с реальными значениями.
- [ ] В трёх файлах `src/lib/i18n/*.json` заполнен `public.legal_notice.provider_body` реальными реквизитами.
- [ ] Открыт `https://<ваш домен>/es/` — WhatsApp-кнопка открывает реальный чат с предзаполненным сообщением.
- [ ] Открыт `https://<ваш домен>/es/aviso-legal/` — там настоящие CIF/NIF и адрес.
- [ ] Футер на всех трёх языках показывает правильные email и телефон.
- [ ] `https://<ваш домен>/sitemap-index.xml` и `https://<ваш домен>/robots.txt` отдаются и содержат правильный домен.

## Требования

- **Node.js 20+** (проверить: `node -v`)
- **npm** (идёт в комплекте с Node)

## Первый запуск

```bash
# 1. Установить зависимости
npm install

# 2. Создать файл .env из шаблона и подставить свои контакты
cp .env.example .env
```

Откройте `.env` и заполните:

```
PUBLIC_CONTACT_WHATSAPP=34663689393         # телефон WhatsApp, только цифры
PUBLIC_CONTACT_EMAIL=hola@spaceforedu.com   # контактный e-mail
PUBLIC_SITE_URL=https://spaceforedu.com     # публичный URL сайта
```

Эти значения подставляются в WhatsApp-кнопки, футер и SEO-ссылки. Без них сайт соберётся, но CTA поведут в пустоту.

## Режим разработки

```bash
npm run dev
```

Откроется `http://localhost:4321/`. Сайт автоматически редиректит на локаль по умолчанию — `/es/`. Изменения в коде подхватываются на лету.

Доступные страницы:

| URL                    | Что |
| ---------------------- | --- |
| `/es/`                 | Главная |
| `/es/homologation/`    | Омологация дипломов |
| `/es/university/`      | Поступление в вуз |
| `/es/spanish/`         | Курсы испанского |
| `/es/pricing/`         | Цены |
| `/es/privacy-policy/`  | Политика конфиденциальности |
| `/es/aviso-legal/`     | Юридическая информация (LSSI-CE) |
| `/es/cookies/`         | Политика cookies |

Замените `es` на `en` или `ru`, чтобы открыть те же страницы на других языках.

## Сборка для продакшена

```bash
npm run build
```

Эта команда делает две вещи:

1. **`astro check`** — проверяет все файлы на ошибки TypeScript и Astro (это и есть текущие автоматические проверки проекта).
2. **`astro build`** — собирает статический сайт в папку `dist/`.

Если `astro check` находит ошибки — сборка остановится, и их нужно починить до деплоя.

## Предпросмотр собранной версии

```bash
npm run preview
```

Запускает локальный сервер на собранных файлах из `dist/` — то же, что увидит пользователь в проде. Полезно перед деплоем.

## Проверки и тесты

В проекте **нет юнит-тестов** — это продающий лендинг, покрывать тестами обычно нечего. Вместо этого используются две проверки:

### 1. Автоматическая проверка типов

```bash
npx astro check
```

Проверяет TypeScript и Astro-файлы на ошибки. Запускается также автоматически при `npm run build`.

### 2. Ручная визуальная проверка

После `npm run dev` пройдитесь по всем страницам и всем трём языкам и убедитесь, что:

- WhatsApp-кнопки открывают `wa.me/<ваш номер>` с нужным текстом.
- Логотипы университетов и фото в Hero-секциях загружаются.
- Переключатель языка (`ES / EN / RU` справа сверху) переводит именно ту страницу, на которой вы сейчас.
- Футер показывает правильный email и телефон.

Если нужна автоматизация — можно добавить Playwright для e2e-тестов; пока не настраивали.

## Деплой на Cloudflare Pages

1. Подключить репозиторий в Cloudflare Pages.
2. Build command: `npm run build`.
3. Build output: `dist`.
4. Добавить те же три переменные окружения (`PUBLIC_CONTACT_WHATSAPP`, `PUBLIC_CONTACT_EMAIL`, `PUBLIC_SITE_URL`) в настройках проекта.

## Структура проекта (коротко)

```
src/
├── pages/[lang]/        ← Astro-страницы, по одной на каждый URL
├── layouts/             ← PublicLayout.astro (обёртка head + navbar + footer)
├── components/
│   ├── layout/          ← Navbar, Footer, моб. меню, переключатель языка
│   ├── pages/           ← React-страницы (Home, Homologacion, ...)
│   ├── public/          ← Переиспользуемые секции (FAQ, отзывы, и т.д.)
│   └── ui/              ← shadcn-компоненты (Button, Card, Dialog, ...)
├── lib/
│   ├── i18n/            ← JSON-файлы переводов (es.json, en.json, ru.json)
│   ├── routes.ts        ← Централизованные ссылки и ссылки на WhatsApp
│   ├── constants.ts     ← ENV-переменные и список локалей
│   └── seo.ts           ← SEO-тексты (title, description) для каждой страницы
└── styles/global.css    ← Tailwind + тема бренда
public/images/           ← Фото, иконки университетов, карты
```

## Как поменять тексты

1. Тексты живут в `src/lib/i18n/es.json`, `en.json`, `ru.json` под ключом `public.<страница>.<ключ>`.
2. Меняете текст в нужном файле → `npm run dev` автоматически покажет изменения.
3. Важно: если добавляете новый ключ, его нужно добавить во все три языка.

## Как поменять контактные данные

Только через `.env` (локально) и через настройки Cloudflare Pages (прод). В коде значения не хардкодятся. тест деплоя.
