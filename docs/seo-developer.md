## Phase 3 — Blog / Content Section 🔴 Critical

This is the highest-ROI task for long-term organic traffic. Each article is a page that ranks for a specific query permanently.

### 3.1 Set up Astro Content Collections

Create `src/content/blog/` with Astro content collections:

```
src/
  content/
    config.ts          ← schema definition
    blog/
      es/
        homologacion-guia-completa.md
      en/
        degree-homologation-spain-guide.md
      ru/
        omologaciya-diploma-v-ispanii.md
  pages/
    [lang]/
      blog/
        index.astro    ← blog listing page
        [slug].astro   ← individual article page
```

### 3.2 Content schema (`src/content/config.ts`)

```ts
import { defineCollection, z } from 'astro:content';

export const collections = {
  blog: defineCollection({
    schema: z.object({
      title: z.string(),
      description: z.string(),
      pubDate: z.date(),
      lang: z.enum(['es', 'en', 'ru']),
      slug: z.string(),
      tags: z.array(z.string()).optional(),
    }),
  }),
};
```

### 3.3 Add Article JSON-LD to blog post pages

```ts
// Add to structured-data.ts
export function blogArticle(params: {
  title: string;
  description: string;
  url: string;
  pubDate: string;
  lang: string;
}): object {
  return {
    "@context": "https://schema.org",
    "@type": "Article",
    headline: params.title,
    description: params.description,
    url: params.url,
    datePublished: params.pubDate,
    inLanguage: params.lang,
    author: { "@id": `${SITE_URL}/#org` },
    publisher: { "@id": `${SITE_URL}/#org` },
  };
}
```

Titles should lead with the keyword, not the brand. Current titles lead with service name — good. Verify each:

## Phase 5 — Performance (Core Web Vitals) 🟡 Important

Google uses Core Web Vitals as a ranking factor. Cloudflare Pages + static Astro is already fast, but verify:

- [ ] Run [PageSpeed Insights](https://pagespeed.web.dev) on `/es/`, `/es/homologation/`, `/es/university/`
- [ ] Target: LCP < 2.5s, CLS < 0.1, INP < 200ms
- [ ] All images: convert to `.webp` format, add `width` and `height` attributes
- [ ] Add `loading="lazy"` to all below-fold images
- [ ] Fonts: use `font-display: swap` in CSS
